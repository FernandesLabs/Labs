## Versioning

Standards follow semantic versioning:
- **Major**: Breaking changes to existing standards
- **Minor**: New standards or non-breaking additions
- **Patch**: Typo fixes, clarifications

## Contributing

To propose a standard change:
1. Create an issue describing the problem
2. Propose changes with rationale
3. Get approval from Lead Engineer
4. Update documentation
5. Create ADR if architectural impact
